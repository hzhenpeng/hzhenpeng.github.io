hexo d上传到github出错的解决办法

发表于 2018-02-26 | 更新于: 2018-09-22 | 分类于 Hexo | 阅读次数: 次
字数统计: 1,031

问题描述

按照hexo官网的 deployment 的流程，执行 hexo d 的时候一直出现fatal: could not read Username for ‘https://github.com‘: No error的错误。。。之前博客还是可以正常执行的，但是这次突然就出问题了。

问题的描述截图如下：

网上查了很多资料，有人说是_config.yml文件格式问题，然后我按照yml格式要求严格书写了一遍，还是不行。然后在SegmentFault上看到使用ssh方式代替https方式上传，自己亲自尝试了一下，成功解决了这个问题。在配置ssh的时候我设置免密deploy。

设置 SSH 使用 hexo deploy 时免输用户名密码

我使用的 hexo 版本是v3.5.0。一开始按照网上的一些教程来设置的时候并没有成功，SSH 需要验证，就是我们键入命令 hexo deploy 以后，当那些需要提交的内容显示完成过后会提示我们输入用户名、密码。我参考了网上的一些教程，现在给出完整可行的设置 SSH 使用 hexo deploy 时免输用户名密码的教程！

生成 SSH 密钥

首先打开命令行工具，通常我们可以直接右键鼠标任意位置打开 Git Bash Here，然后输入命令cd ~/.ssh，检查”.ssh”目录是否在你的用户目录中，然后在本地计算机生成密钥：

使用默认的文件名直接enter，使用默认的密码（空）直接enter，密钥生成成功

设置 Github 项目的 Deploy keys

可以选择在用户设置中添加密钥（Settings –> SSH GPG keys）还是单个项目中添加密钥，在这里我就在我们的博客项目中添加密钥，此处以我的博客为例。
在自己的github的首页菜单栏中点击 Settings –> Deploy keys –> Add deploy key，然后将生成的 id_rsa.pub 中的内容全选复制到 key 输入框中，然后点击 Add key 完成添加公钥。

注意，这里要勾选Allow write access，否则后面会出错！

接下来测试配置是否成功

如果出现以下内容即表示配置完成并且成功！

设置博客的提交地址

到了这一个步骤和大家在网上搜到的其他教程就不一样了，网上大多数教程这一步的设置

但是我根据网上的并没有用，而且我的 blog 文件夹中包含 .git 文件夹的文件夹名字是 .deploy_git，不是.deploy。还有我输入 $ git remote -v 时的显示结果为空，我尝试过自己添加 $ git remote add origin git@github.com:username/username.github.io.git,然后 $ hexo deploy 还是需要输入密码。
在 hexo deploy 的控制台输出结果中我看到了链接仍然是 https 的链接，这时我想到了 hexo 的配置文件 _config.yml 中的 deploy 属性。

解决办法就是

我将其中的 repository修改成了git@github.com:username/username.github.io.git，这个时候我再尝试了 $ hexo deploy 果然成功了，就不用输入密码了！之前hexo deploy的出错问题也解决了，并且用ssh方式代替了https方式。

到此我的解决方法已经完成了，希望能帮大家！
